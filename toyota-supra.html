Project Case Study // Vehicle Pipeline

Supra MK4 3D Model

High-detail Toyota Supra MK4 modeling, rigging, and Unreal Engine 5 integration

This project is my 3D interpretation of the Toyota Supra MK4, created using reference images to match proportions, shapes, and iconic design elements. I focused on achieving clean hard-surface topology, accurate curves, and a complete structure that includes both the exterior and interior components. The goal was not just visual accuracy, but creating a model that is efficient, smooth, and suitable for real-time use. Working on this car helped me refine my workflow for automotive modeling, improve my edge-flow decisions, and better understand how to prepare high-detail assets for game engines.

Technical Implementation

Unreal Engine 5 Integration

After finishing the model, I brought it into Unreal Engine 5 and set it up using the Chaos Vehicle System to make the Supra fully drivable. I configured wheel setups, suspension behavior, mass distribution, friction curves, and engine settings to achieve a realistic driving feel. This step taught me how vehicle physics behave inside Unreal Engine, how different parameters affect handling, and how to optimize a model for real-time movement. Making the car drivable gave me practical experience in technical implementation, physics tuning, and gameplay-focused asset integration.

Preparation Pipeline

Rigging and Vehicle Setup

Before bringing the car into Unreal Engine, I ensured the model was prepared with a clean and functional rig. I rigged the wheels, steering, and essential movable parts directly inside the software, using built-in tools without relying on any external plugins. This included setting up pivot points, wheel rotations, steering axes, and hierarchy clean-up to make the mesh ready for animation and engine import. The rigging process helped me understand how vehicles should be structured for game engines and how proper rig preparation directly improves the in-engine driving experience.
